# 智能垃圾分类小程序 - 陈氏模型 ER 图设计

## 一、实体（矩形）及属性（椭圆）

### 1. Users（用户实体）
```
┌─────────────────────────────────────┐
│              Users                  │
├─────────────────────────────────────┤
│  PK │ id（主键）                    │
│     │ openid（微信ID）              │
│     │ nickname（昵称）              │
│     │ avatar_url（头像）            │
│     │ total_points（总积分）        │
│     │ check_in_days（连续签到天数） │
│     │ identify_count（识别次数）    │
└─────────────────────────────────────┘
```

### 2. trash_categories（垃圾分类实体）
```
┌─────────────────────────────────────┐
│        trash_categories             │
├─────────────────────────────────────┤

│     │ name（垃圾名称）              │

│     │ type_name（分类名称）         │
│     │ tips（投放提示）              │
```

## 二、关系（菱形）及基数

```
┌─────────────────────────────────────────────────────────┐
│                      关系定义                            │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   Users ◆──────<identifies>──────◇ trash_categories    │
│         1         N                                   │
│   （一个用户可以识别多种垃圾）                          │
│                                                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   Users ◆──────<has_history>─────◇ identify_history    │
│         1         1                                   │
│   （一个用户有多条识别记录）                            │
│                                                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   Users ◆──────<exchanges>───────◇ prizes              │
│         1    N    M         1                         │
│   （多对多：通过 exchange_records 关联）                │
│                                                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   Users ◆──────<achieves>────────◇ achievements        │
│         M    N                                        │
│   （多对多：通过 user_achievements 关联）               │
│                                                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   Users ◆──────<favorites>───────◇ trash_categories    │
│         M    N                                        │
│   （多对多：用户收藏垃圾信息）                          │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## 三、在 Draw.io 中绘制步骤

### 步骤 1：放置实体（矩形）
在画布上均匀分布放置 7 个矩形：
```
┌──────────┐      ┌──────────┐      ┌──────────┐
│  Users   │      │  prizes  │      │  trash_  │
│          │      │          │      │categories│
└──────────┘      └──────────┘      └──────────┘

┌──────────┐      ┌──────────┐      ┌──────────┐
│ identify │      │ exchange │      │ recycle_ │
│ _history │      │ _records │      │  points  │
└──────────┘      └──────────┘      └──────────┘

       ┌──────────┐
       │achieve-  │
       │  ments   │
       └──────────┘
```

### 步骤 2：添加属性（椭圆）
每个实体下方放置椭圆属性，用线连接：
- 主键属性标注 **PK**
- 外键属性标注 **FK**

### 步骤 3：添加关系（菱形）
在有关联的实体之间放置菱形：
```
     identifies
        ◇
       / \
      /   \
   Users   trash_categories
```

### 步骤 4：标注基数
在线上标注 1:N、1:1 或 M:N：
```
   1         N
Users ◆───────◇ trash_categories
```

## 四、陈氏模型符号说明

| 符号 | 含义 | 用途 |
|:----:|:----:|:----:|
| ▭ 矩形 | 实体 | 表示数据表 |
| ○ 椭圆 | 属性 | 表示字段/列 |
| ◇ 菱形 | 关系 | 表示实体间的关联 |
| ─ 线 | 连接 | 连接实体与属性/关系 |
| 1,N,M | 基数 | 表示数量关系 |
| PK | 主键 | 唯一标识 |
| FK | 外键 | 外键关联 |
